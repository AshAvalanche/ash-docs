Avalanche Transactions

In this section, we will learn how to use the ash.avalanche.tx module of the Ansible Avalanche Collection to submit transactions to an Avalanche network.

note

For this tutorial, we will use the local inventory provided by the Ansible Avalanche Getting Started repository. See Create a Local Test Network for details.

Perform an AVAX transfer from the X-Chain to the P-Chain​

The notebook ash.avalanche.transfer_avax is provided as an example. We will call this notebook and follow best practices to secure Avalanche keystore credentials using Ansible vault:

  1. In inventories/local/group_vars/api_node.yml, the keystore crendentials have been encrypted using Ansible vault:

      6138
    avax_transfer_from_password: !vault |
      $ANSIBLE_VAULT;1.1;AES256
      36363962333839616366363730306264373332373336316332356162323230313039393265626639

    This avoids storing usernames and passwords in plain text.

  2. To run the notebook, we need to provide the vault password (ewoq):

    ansible-playbook ash.avalanche.transfer_avax -i inventories/local --ask-vault-password

  3. The notebook issues 2 transactions to transfer 1 AVAX from the X-Chain to the C-Chain and you can see the tx module output format:
